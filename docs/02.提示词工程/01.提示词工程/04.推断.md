## 核心定义：什么是“推断”？

核心思想：模型从给定的文本中理解潜在含义、提取特定属性或判断意图。它不仅是阅读，更是“解读”。

## 推断的五大典型应用

文章通过一个“台灯评论”的例子，展示了模型如何像人类专家一样分析文本。

### 情感分析 (Sentiment Analysis)

- 做法：让模型判断文本的情绪。
- 示例：以下评论的情感是什么？（正面/负面）

- Agent 场景：自动监控社交媒体，第一时间发现并预警用户的差评。


### 识别情绪类型 (Identifying Emotions)

- 做法：让模型识别具体的心理状态（如：愤怒、开心、沮丧等）。
- 示例：这篇评论是否表达了“愤怒”？请回答是或否。

- Agent 场景：客服 Agent 可以优先处理那些“极其愤怒”的客户请求。


### 提取信息 (Information Extraction)

- 做法：从非结构化文本中抽取出特定的实体或属性。
- 示例：提取评论中的产品名称和公司名称。

- 示例输出：{"item": "台灯", "brand": "Lumina"}

- Agent 场景：自动从海量邮件中提取订单号、快递号，并录入数据库。


### 主题推断 (Topic Inferring)

- 做法：判断一段长文在讲什么，或者是否包含预设的主题。
- 示例：这段文字在讨论以下哪些话题：政府、航空、社会、科技？

- Agent 场景：新闻自动分类系统，将文章分发给不同的频道。


### 多任务并行 (Multi-tasking)

- 做法：在一个 Prompt 里让模型完成上述所有工作。

- 示例：

  请分析评论：

  1. 情感极性；
  2. 是否愤怒；
  3. 提取商品名；
  4. 提取公司名。请以 JSON 格式输出。

## 为什么“推断”对写 Agent 至关重要？

作为从 0 到 1 的开发者，你必须理解“推断”是 Agent 逻辑判断的来源：

- 实现“分流器”功能： 你的 Agent 接收到用户的一句话，首先需要通过“推断”来判断用户的意图（意图识别）。

  - 推断结果 A：用户想买东西 -> 触发购物逻辑。


  - 推断结果 B：用户想投诉 -> 触发售后逻辑。


- 降低开发成本： 以前需要五个模型做的事，现在一个 Prompt 模版搞定，且准确率极高。

- 零样本能力 (Zero-shot)： 即使你没有提前给模型看过任何关于你产品的评论，它也能凭借通用知识准确推断。


## 初学者实战建议

1. 首选 JSON 输出：在推断任务中，务必强制要求模型输出 JSON。这样你的 Python 代码才能直接用 json.loads() 读取结果，并据此执行不同的 if-else 逻辑。
2. 给模型候选名单：在提取标签或判断情感时，给模型一个范围（如：从 [开心, 中立, 难过] 中选择），这比让它随意发挥要稳定得多。

3. 组合拳：先用第 4 章的“概括”缩小文本范围，再用这一章的“推断”提取核心信息。
