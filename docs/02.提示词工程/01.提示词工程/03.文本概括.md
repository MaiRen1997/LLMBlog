## 核心任务：文本概括的本质

核心思想：概括不仅仅是简单的“精简”，而是根据业务需求对信息进行有目的的过滤和重组。

## 核心技巧：多维度的概括方法

文章通过“电商产品评论”的案例，展示了四种不同的概括侧重点：

### 限制字数或句数（基础控制）

- 技巧：在 Prompt 中明确要求“用最多 30 个词总结”或“用 2 条短语概括”。

- 示例：请用一句话总结以下评论内容：...


- Agent 场景：用于生成摘要预览，防止 UI 界面被长文本撑破。


### 侧重特定角度（关键点过滤）

- 技巧：明确告诉模型你关心什么。例如侧重于“物流”或“价格”。
- 示例：请根据以下评论生成摘要，以便给物流部门反馈，重点关注运输速度和包装。

- 示例对比：

  - 侧重物流：会提到“快递很快，没破损”。

  - 侧重价格：会提到“性价比高，折扣给力”。


### 提取而非概括（关键信息挖掘）

- 技巧：使用“提取（Extract）”关键词而非“总结（Summarize）”。
- 示例：从以下评论中提取出快递公司名称和送达天数。

- Agent 意义：这是将非结构化文本转为数据库字段的核心技术。


##  规模化处理（批量概括）

核心思想：当你有 100 条评论时，手动总结是不可能的。Agent 的威力在于自动化循环。

- 流程：

  1. 建立一个评论列表。

  2. 编写一个通用的 Prompt 模版。

  3. 利用 for 循环遍历列表，调用模型。

  4. 将所有总结结果汇聚成一个总表。


## 对初学者的 Agent 开发启示

作为从 0 到 1 的开发者，你可以从这篇文章中获得以下进阶思路：

### “概括”是 Agent 的信息过滤器

- 如果你的 Agent 正在处理一个超长文档（超过了模型的上下文长度），你可以先让 Agent 把每一段概括一下，再对总结后的内容进行二次处理。

### “侧重点”是 Agent 的专业分工

- 你可以创建两个 Agent：
  - 售后 Agent：专门“提取”评论中的产品缺陷。

  - 物流 Agent：专门“提取”评论中的配送问题。

  - 它们使用同一个原始文本，但因为 Prompt 的侧重点不同，产出的结果各司其职。


### 避免信息丢失

注意：在“提取”任务中，如果提示词写得不好，模型可能会漏掉关键信息。务必结合前两章提到的迭代优化，检查模型是否真的提取到了你需要的字段。
