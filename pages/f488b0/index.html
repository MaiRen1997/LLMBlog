文本扩展

# 核心定义：什么是扩展？

文本扩展是将短文本（如一组关键词或一个简短的主题）作为输入，由大模型生成更长的文本（如一封完整的邮件或一篇文章）。

注意：这与“概括”相反。扩展能让 Agent 具备从“意图”到“执行”的落地能力。

# 核心案例：自动回复客户评论

文章通过一个具体的场景展示了如何利用模型生成个性化回复：

# 动态逻辑：根据情感定制内容

  • 步骤一：先利用“推断（Inferring）”技术识别客户评论的情绪。

  • 步骤二：根据情绪编写不同的指令。

    • 如果是正面反馈：表达感谢，并询问是否愿意分享。
    • 如果是负面反馈：表示歉意，并提供后续处理方案。

  • 示例：

    • "根据以下客户评论写一封感谢信。如果评论是积极的，回复要充满热情；如果评论是消极的，回复要诚恳道歉。"

# 事实约束：减少幻觉 (Hallucination)

  • 技巧：在 Prompt 中包含“事实资料库”（如产品说明书）。
  • 作用：要求模型仅基于提供的资料进行回复，防止模型编造不存在的功能或价格。

# 关键参数：Temperature (温度)

这是本章最重要的技术点。它决定了模型在扩展时的“想象力”边界。

  • 低温度 ($T \approx 0$)：

    • 特点：模型总是选择概率最高的词。
    • 优点：结果稳定、可靠、逻辑严密。
    • 场景：写代码、数据分析、基于事实的回复。

  • 高温度 ($T > 0.7$)：

    • 特点：模型会尝试一些概率较低的词。
    • 优点：结果更多样、有创意。
    • 场景：创意写作、头脑风暴。

# 对初学者的 Agent 实践建议（三层架构）

当你从 0 到 1 写 Agent 时，本章提供的逻辑可以组成 Agent 的 “表达层”：

  1. 输入层：收集用户的反馈或数据。

  2. 处理层：结合 事实单 + 用户评论。

  3. 生成层：

    • 设定 以保证稳健性。

    • 明确受众和语气（如“回复要像专业客服一样”）。

# 知识点汇总对照表

核心要素 实践要点 备注
个性化 根据输入（如情绪）改变指令 使 Agent 更有“人味”
事实性 提供背景资料（Context） 解决 AI 乱说话的问题
多样性 调整参数 Temperature 平衡逻辑与创意
道德规范 明确告知用户内容由 AI 生成 防止误导，符合伦理
