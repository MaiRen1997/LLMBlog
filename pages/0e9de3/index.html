# 核心定义：什么是“文本转换”？

核心思想：将输入文本按照特定规则，改变其表达形式（如语言、风格、格式），但保持其核心含义不变。

# 六大核心转换场景

文章通过具体的代码示例展示了 LLM 在处理文本格式和风格时的灵活性。

# 语言翻译 (Translation)

  • 多语言处理：可以同时翻译成多种语言，或者识别文本属于哪种语言。

# 语气/风格转换 (Tone Transformation)

  • 做法：根据受众调整说话的腔调。

  • 示例：将这段非正式的邮件改写为正式的商务邮件。

  • Agent 场景：你的 Agent 可以根据客户的会员等级，自动切换“热情活泼”或“专业稳重”的回复风格。

# 格式转换 (Format Conversion)

  • 做法：在 JSON、HTML、XML、Markdown 等各种结构化数据之间相互转换。

  • 示例：将这个 Python 字典转换为 HTML 表格，包含标题和内容。

# 拼写与语法纠错 (Spellcheck & Grammar Check)

# 为什么“转换”对写 Agent 至关重要？

作为初学者，你需要意识到 Agent 本质上是一个信息流转站。转换功能解决了信息传递过程中的三大痛点：

  1. 多语言适配（Internationalization）
    1. 你的 Agent 只要能看懂一种语言，就能通过“转换”功能服务全世界的用户。
  2. 数据标准化（Data Normalization）
    1. Agent 从网页上抓取的信息往往是乱七八糟的，通过“转换”功能，可以统一成你的程序能读懂的 JSON 格式。
  3. 用户体验优化（Personalization）
    1. Agent 不需要针对每个场景写死回复模板，而是通过“转换”动态生成符合当前语境的文本。

# 初学者实战技巧

  1. 使用“Few-shot”增强效果：如果你对转换后的语气有严格要求，给模型一个“改前”和“改后”的例子（见第 2 章）。

  2. 防止过度修改：如果你只想纠错而不希望模型改变你的原意，可以加上指令：“请仅修改拼写和语法错误，尽量保持原句的结构和词汇。”

  3. Debug 建议：转换 HTML 或 JSON 时，有时模型会多写一些解释。记得加上：“只输出转换后的代码，不要输出任何解释。”
